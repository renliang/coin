# 币种筛选系统设计文档

## 目标

构建一个自动化筛选工具，从全市场小市值币种中找出呈现"底部蓄力"形态的币种：缩量、缓慢下跌、幅度不大、持续1-2周。

## 方案

CoinGecko 免费API初筛市值 → Binance K线做形态检测 → CLI输出结果。纯筛选器，不入库，手动触发。

### 模块职责

1. **CoinGecko 筛选器** (`scanner/coingecko.py`) — 拉取币种列表，按市值过滤（默认<1亿美元），输出符合条件的币种及其 Binance 交易对映射
2. **K线拉取器** (`scanner/kline.py`) — 用 ccxt 从 Binance 拉日K线（最近30天），处理API限速（CoinGecko 30次/分钟，Binance无严格限制但需适当间隔）
3. **形态检测引擎** (`scanner/detector.py`) — 对每个币种的K线数据跑四项筛选条件
4. **评分与排序** (`scanner/scorer.py`) — 四项条件各给0-1分，加权求和，按总分降序排列
5. **CLI 入口** (`main.py`) — 串联流程，tabulate输出表格

## 形态检测逻辑

检测窗口：最近7-14天（可配置）。四个条件全部满足才算命中：

### 1. 缩量判断

将检测窗口分为前半段和后半段。后半段均量 < 前半段均量 × `volume_ratio`（默认0.5）。

### 2. 下跌趋势

窗口内收盘价序列的线性回归斜率为负。

### 3. 跌幅范围

窗口内从最高收盘价到最低收盘价的跌幅在 `drop_range` 范围内（默认5%-15%）。

### 4. 缓慢确认

窗口内所有交易日的单日涨跌幅均不超过 ±`max_daily_change`（默认5%），排除暴涨暴跌。

## 评分机制

| 维度 | 计算方式 | 权重 |
|------|---------|------|
| 缩量程度 | `1 - (后期均量/前期均量)`，clamp到[0,1] | 0.3 |
| 跌幅温和度 | 跌幅越接近区间中心(10%)得分越高 | 0.25 |
| 趋势稳定性 | 回归R²值（越接近1越稳定） | 0.25 |
| 波动平稳度 | `1 - (最大单日涨跌幅 / max_daily_change)`，clamp到[0,1] | 0.2 |

总分 = 各维度得分 × 权重之和，范围[0, 1]。

## 可配置参数

```yaml
scanner:
  max_market_cap: 100000000    # 市值上限（美元），默认1亿
  window_min_days: 7           # 检测窗口最小天数
  window_max_days: 14          # 检测窗口最大天数
  volume_ratio: 0.5            # 缩量阈值（后期量/前期量）
  drop_min: 0.05               # 最小跌幅
  drop_max: 0.15               # 最大跌幅
  max_daily_change: 0.05       # 单日最大涨跌幅
  top_n: 20                    # 输出前N个结果
```

```bash

```

## API限速处理

- **CoinGecko 免费版：** 30次/分钟。币种列表分页拉取，每页250条，每次请求间隔2秒。
- **Binance K线：** 无严格限制，但每次请求间隔0.5秒防止被封。
- 预计扫描全量小市值币种（约几百个）耗时5-10分钟。
